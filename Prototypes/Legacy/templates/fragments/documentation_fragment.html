Instructions

This prototype keeps the legacy documentation layout so we can test readability, table-of-contents behavior, and page density before the real system exists.

What This Prototype Covers

The shell, page panes, alerts, file views, maintenance layout, and settings forms are all simulated using browser-side state.

Questions To Pressure-Test

Multi-world visibility

Should the home dashboard remain single-world focused, or should it become a world switcher and summary page?

Privilege surfaces

Which actions need modal confirmation, inline confirmation, or both when multiple worlds exist at the same time?

Cluster readiness

If worlds eventually live on different machines, which of these pages should show host ownership explicitly?

Next UI Iteration Targets

  • Multi-world list page
  • Per-world address and version badges
  • More explicit alert severity model
  • Cleaner file-management affordances
